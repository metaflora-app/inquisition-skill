# Metaflora Inquisition v0.1.0 Release Notes

## Релизная рамка

`v0.1.0` — первый публичный релиз `metaflora-inquisition`, skill для проверки утверждений через процедуру трибунала.

Релиз собирает в одном пакете:

- сам skill;
- готовый `.skill`-архив;
- полный release bundle;
- README-манифест;
- установочную инструкцию;
- GitHub Release body;
- checksum-и;
- три скетчноута;
- smoke-тесты.

## Предисловие к релизу

Верификация в агентных workflows часто страдает от вежливости. Пользователь просит "проверить", агент ищет пару совпадающих страниц, выдаёт аккуратный вывод, затем задача уходит дальше: в продуктовый документ, статью, pitch deck, пост, аналитику или решение по roadmap.

`metaflora-inquisition` вводит другой режим работы: утверждение проходит разбирательство.

Проверяется происхождение, источник, логика, контекст, область применимости, конфликт интересов и уровень неопределённости. Когда выводы расходятся, трибунал выбирает одну точку разлома и назначает давление. Финал оформляется как булла с одним вердиктом.

## Что нового

### Полная пятичастная процедура

Релиз включает весь процесс:

```text
ADMONITIO -> INTERROGATIO -> DECISIO TRIBUNALIS -> TORTURA -> BULLA
```

Каждая стадия имеет назначение, формат входа, формат выхода и условия перехода.

### Независимые инквизиторы

На `INTERROGATIO` работают три роли:

- `Fontium` — источники;
- `Rationis` — логика;
- `Contextus` — контекст.

Они проверяют разные плоскости тезиса. Это снижает шанс, что живой URL будет принят за доказательство всего вывода.

### Решение трибунала

Трибунал получает выводы инквизиторов и выбирает маршрут:

- сразу к булле;
- через `TORTURA`.

Он фиксирует причину, точку удара и предварительный вердикт, если давление не требуется.

### Точечная Tortura

`TORTURA` проводится только по одной конкретной точке. В ней сталкиваются:

- `Advocatus Diaboli`;
- `Advocatus Fidei`.

Один атакует тезис, второй защищает тезис. После этого трибунал фиксирует победителя в точке.

### Финальная Bulla

Булла — главный output релиза. Она содержит:

- предмет разбирательства;
- статус происхождения;
- выводы трёх проверок;
- решение по `TORTURA`;
- финальный вердикт;
- оговорки;
- исправленную версию при необходимости;
- проверенные источники;
- уровень доверия.

## Вердикты

Релиз поддерживает пять вердиктов.

### ABSOLVO

Утверждение прошло проверку. Источник живой, логика держится, контекст сохранён.

### ABSOLVO CUM NOTA

Утверждение пригодно с существенными оговорками. Пользователь получает границы применимости.

### PENITENTIA

В утверждении есть пригодное ядро, формулировка требует правки. Пользователь получает исправленную версию.

### ANATHEMA

Утверждение противоречит проверенным данным. Пользователь получает корректную замену.

### IGNORAMUS

Данных недостаточно. Пользователь получает честную фиксацию неопределённости.

## Сценарии применения

### Проверка статистики

Пример:

```text
Reels с субтитрами получают на 40% больше охвата
```

Проверяются источник цифры, период, метрика, причинность, рынок, первичный отчёт и цепочка пересказов.

### Проверка цитат

Пример:

```text
Автор X сказал, что AI заменит 80% аналитиков
```

Проверяются оригинальная публикация, точность цитирования, контекст и дата.

### Проверка benchmark

Пример:

```text
Модель X лучше модели Y в кодинге
```

Проверяются версия модели, дата сравнения, тип задач, benchmark, независимость источника и переносимость вывода.

### Проверка прогнозов

Пример:

```text
Рынок AI-агентов достигнет 50 млрд долларов к 2030 году
```

Проверяются методология, базовый год, CAGR, рыночные границы, сценарии и ключевые неопределённости.

### Проверка output другого AI

Пример:

```text
Другой агент написал, что закон уже вступил в силу
```

Проверяются официальный источник, дата, юрисдикция, актуальная редакция и применимость.

## Известные ограничения

- Веб-поиск нужен для полной проверки источников и контекста.
- Закрытые данные требуют доступа к внутренним источникам.
- Слишком широкие тезисы нужно разложить на атомарные утверждения.
- Прогнозы оцениваются по обоснованности, горизонту и неопределённостям.
- Результаты веб-проверки нужно датировать при использовании в публичных материалах.
